# Code Of Conduct

## Our Standard

This project is for technical collaboration around open-source engineering software. Participation requires respectful, evidence-driven behavior.

Examples of expected behavior:

- explain technical disagreement with concrete reasoning
- document assumptions and limitations clearly
- accept review feedback without hostility
- help newcomers navigate the repo structure and contribution rules

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- repeated bad-faith arguments
- deceptive benchmark or validation claims
- publishing private information without permission

## Scope

This code of conduct applies in repository issues, pull requests, discussions, and related project communication.

## Enforcement

Project maintainers may remove comments, close discussions, or block participation for behavior that violates this policy.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers through the repository owners' preferred channel. If no private channel is listed yet, open a minimal public issue requesting moderator contact without posting sensitive details.
